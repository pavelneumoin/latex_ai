**Дата:** 2026-05-05

---

## 1. Цель проекта

Сделать веб-сервис, на котором учитель за **1–2 минуты** получает готовый к печати PDF рабочий лист по математике или информатике — либо из своих материалов, либо по запросу темы. Плюс автопроверка заполненных работ по числовым ответам.

**Главное обещание:** *«учитель приходит на сайт, описывает урок словами, нажимает «сгенерировать», получает PDF с его лого и водяным знаком, печатает, раздаёт детям. Потом фотографирует/сканирует, загружает обратно — нейросеть выставляет оценки.»*

---

## 2. Целевая аудитория

- Учителя математики и информатики 5–11 классов (РФ).
- Уровень владения IT — от «открыть Word» до «знаю Wildcat и LaTeX».
- Сценарий — нужен лист на завтра, на сегодня, на следующий урок. Время на подготовку — **5–15 минут**.
- Важно: **никакого LaTeX-кода глазами учителя не видит**. Только UI.

## 3. Пользовательские сценарии

### Сценарий А — «По теме»
1. Учитель: «Производная. 11 класс. 12 заданий. Без теории. С клеткой. Ответы на отдельной странице».
2. Сервис: 30–60 сек генерации → PDF с лого учителя на скачивание.
3. Опция «сохранить в моих листах» / «опубликовать в каталоге».

### Сценарий Б — «Из материала»
1. Учитель загружает PDF учебника / фотку доски / Word с задачами.
2. Сервис извлекает условия, переформатирует под шаблон, добавляет оформление.
3. Тот же выход — PDF.

### Сценарий В — «Проверка»
1. Учитель скачал/раздал лист. Дети заполнили.
2. Загружает обратно — PDF/JPG/несколько файлов.
3. Сервис распознаёт ответы в пронумерованных полях, сравнивает с эталоном, выдаёт **таблицу оценок** + PDF с разметкой ✓/✗.

### Сценарий Г — «Каталог» (позже)
- Витрина чужих листов: фильтр по классу/теме/типу. Скачать бесплатно или купить.

## 4. Функциональные требования

### MVP (v0.1) — 4 недели

**Генератор:**
- Один шаблон «Универсальный А4».
- Поля формы: предмет (мат/инф), класс (5–11), тема (свободный текст), число задач (1–20), теория (нет / без пропусков / с пропусками), клетка (да/нет), ответы (нет / в конце / отдельной страницей).
- Бренд: загрузка PNG-лого + текст водяного знака.
- LLM собирает контент → шаблонизатор LaTeX → pdflatex → PDF на скачивание.

**Storage:**
- Каждая генерация сохраняется в БД с метаданными.
- Личный кабинет «Мои листы» — список с превью.

**Auth:** магическая ссылка на email (без паролей).

**Что НЕ входит в MVP:**
- Каталог, проверка, маркетплейс, мобильное приложение, OCR.

### v0.2 — Проверка (через 4 недели после MVP)

- Кнопка «Проверить работу» на странице листа.
- Загрузка PDF/JPG. Vision-LLM (Claude Sonnet) читает поля ответов, сравнивает с эталоном.
- Таблица: ученик / задача 1 / задача 2 / … / итоговая оценка по 5-балльной шкале (по правилам пользователя из mesh-marks: ≥86%→5, 50–85→4, <50 не ставим).

### v0.3 — Шаблоны и кастомизация
- 3–5 шаблонов: универсальный, ОГЭ-стиль, ЕГЭ-стиль, контрольная, самостоятельная.
- Редактор «вручную править / добавлять задачи».
- История версий листа.

### v1.0 — Каталог и публикация
- Публичная витрина листов.
- Подписка / разовая оплата (определим позже).
- Revenue share с авторами листов.

## 6. LLM-стратегия

### Раундирование моделей

| Задача                              | Модель                                      | Цена (за 1 ученика) |
|-------------------------------------|---------------------------------------------|---------------------|
| Генерация задач по теме (короткие)  | Claude Haiku 4.5 (через Anthropic API)      | ~₽0.20              |
| Генерация контрольной (длинная)     | Claude Sonnet 4.6                           | ~₽1.50              |
| Vision: проверка работы             | Claude Sonnet 4.6 (vision)                  | ~₽2.00 за работу    |
| Резерв / fallback                   | OpenRouter (GPT-4o-mini, Gemini Flash)      | ~₽0.10              |
| Российский трафик                   | GigaChat Lite                               | бесплатно (лимит)   |

**Решение:** на MVP — только Claude Haiku/Sonnet через Anthropic API напрямую (не OpenRouter). Это:
- ниже latency,
- prompt caching из коробки (важно — шаблоны промптов будут одинаковые),
- стандартный Anthropic SDK, у тебя уже есть опыт.

OpenRouter и GigaChat — добавим в v0.3, когда будем оптимизировать стоимость.

### Промпт-шаблоны
- `prompts/generate_math_tasks.md` — для математики
- `prompts/generate_cs_tasks.md` — для информатики
- `prompts/check_worksheet.md` — для vision-проверки
- `prompts/extract_from_pdf.md` — для сценария «из материала»

Все промпты — версионируем в git, кэшируем (cache_control system block).

## 7. Шаблоны рабочих листов

### MVP — один шаблон «WS-Universal»

Структура (на основе твоих feedback_worksheet_design.md и feedback_worksheet_layout.md):

```
[Шапка: лого + ФИО + класс + дата]
[Раздел 1: Теория (опционально, с пропусками или без)]
[Раздел 2: Классная работа — задачи 1..N]
[Раздел 3: Самостоятельная — 1 колонка (правило!)]
[Раздел 4: Домашка (опционально)]
[Раздел 5: Ответы (на отдельной странице или скрыты)]
[Подвал: водяной знак / QR на учителя]
```

Параметризация через JSON:
```json
{
  "subject": "math",
  "grade": 11,
  "topic": "Производная",
  "task_count": 12,
  "theory_mode": "with_gaps",
  "grid": true,
  "answers": "separate_page",
  "branding": {
    "logo_url": "s3://.../logos/user42.png",
    "watermark_text": "© Иванов И. И., 2026"
  }
}
```

Бэкенд берёт этот JSON → генерирует LaTeX-источник → компилирует → возвращает PDF.

### v0.3 — расширение
- WS-Контрольная (5 вариантов параллельно)
- WS-Самостоятельная (10 минут, 4 задачи)
- WS-ОГЭ-инф (типовая структура заданий №1–13)
- WS-ЕГЭ-мат-проф (типовая структура №1–18)
- WS-Тренажёр (одна тема, 30+ задач, без теории)

## 8. Брендинг и кастомизация

- **Лого:** PNG/SVG, ≤500КБ, в шапке слева.
- **Водяной знак:** текст или картинка, центр листа, 10% opacity.
- **QR-код:** опционально, ссылка на учителя/группу.
- **Цвет акцента:** один цвет на лист (по умолчанию синий, можно поменять).
- **Подпись:** «© ФИО, год» в подвале.

Хранится в профиле учителя — не нужно загружать каждый раз.

## 9. Проверка работ (чекер)

### Принцип

LaTeX-шаблон рисует **пронумерованные поля для ответов** в фиксированных областях. Например:
```
[Задача 5]
... условие ...
Ответ: ┌─────────┐ [#5]
       └─────────┘
```

При генерации сохраняем:
- координаты каждого поля,
- эталонный ответ (число / список чисел / слово).

### Процесс проверки

1. Учитель загружает PDF/JPG (сканы или фото).
2. Vision-LLM получает картинку + JSON координат + JSON эталонов.
3. Промпт: «прочитай содержимое поля #N, сравни с эталоном `42`, верни `correct`/`wrong`».
4. Бэкенд считает % правильных, выставляет оценку.
5. UI показывает таблицу + размеченный PDF (зелёные/красные кружки).

### Ограничения MVP-чекера

- Только числовые ответы (5; -3.14; 100).
- Допуск: точное совпадение или ±0.01 (настраивается).
- Не работает с развёрнутыми решениями.

### Расширения v1.0
- Проверка коротких слов (да/нет, имена, термины).
- Сравнение «по смыслу» через LLM (для развёрнутых).

## 13. Риски

| Риск                                                  | Вероятность | Митигация                                           |
|-------------------------------------------------------|-------------|-----------------------------------------------------|
| LLM генерит математически некорректные задачи         | высокая     | прогон через Python-валидатор (как в z16-серии)     |
| Vision-чекер ошибается на детском почерке             | высокая     | начать с печатных ответов; пилот на 1 классе        |
| LaTeX-компиляция падает на минорных багах             | средняя     | sandboxing + retry + fallback на простой шаблон     |
| Стоимость LLM выше прогноза                           | средняя     | prompt caching + переход на дешёвые модели          |
| Учителя не понимают UI                                | средняя     | юзабилити-тест на 3–5 учителях после макетов        |
| Конкуренты (Решу.ОГЭ, Школково, ЯКласс) запустят то же| низкая      | у нас есть готовая Wildcat-вёрстка — это преимущество|
