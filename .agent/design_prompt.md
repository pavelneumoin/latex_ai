# Design prompt для РабочийЛист.ai

> Скопируй и вставь это целиком в Claude Design (или v0.dev / Lovable / Figma AI) — на выходе должен быть готовый набор экранов.

---

## ЗАДАЧА

Спроектируй UI веб-приложения **РабочийЛист.ai** (стилизация: `РабочийЛист·ai` или `рабочийлист/ai`) — сервис для русских учителей математики и информатики, где они за минуту получают готовый PDF рабочий лист. Целевая аудитория — учителя 25–55 лет, многим не близок IT — UI должен быть **очень дружелюбным и понятным с первого взгляда**.

Стек реализации: **Next.js 15 + TypeScript + Tailwind CSS + shadcn/ui**. Дай готовые React-компоненты на этом стеке.

---

## БРЕНД

- **Тон:** профессиональный, но не сухой. Дружелюбный, но не игривый.
- **Имя:** РабочийЛист.ai (стилизация в логотипе: `РабочийЛист·ai` или `рабочийлист/ai`, в коде/URL — `rabochiilist`).
- **Логотип:** придумай простой геометрический знак — лист бумаги + искра/звезда, или абстрактная сетка с ячейкой. Чёрный/тёмно-синий + один акцентный цвет. В нейминге используй кириллический шрифт-гротеск.
- **Слоган:** «Рабочий лист за минуту. Печатай и веди урок.»

### Палитра
- **Primary:** глубокий синий `#1E40AF` (Tailwind `blue-700`)
- **Accent:** тёплый янтарный `#F59E0B` (Tailwind `amber-500`) — только для CTA и важных акцентов
- **Surface:** белый `#FFFFFF` и очень светлый серый `#F8FAFC` (`slate-50`)
- **Text:** `#0F172A` (`slate-900`), вторичный — `#64748B` (`slate-500`)
- **Success:** `#10B981` (`emerald-500`)
- **Error:** `#EF4444` (`red-500`)
- Минимум градиентов. Тени мягкие, `shadow-sm`/`shadow-md`. Скругления `rounded-2xl` для карточек, `rounded-lg` для кнопок.

### Типографика
- Заголовки: **Manrope** или **Inter**, weights 600–700.
- Текст: **Inter**, weight 400–500.
- Размеры: h1 36px, h2 28px, h3 20px, body 16px, caption 14px.
- Не используй курсив без нужды. Не используй мелкий текст <14px.

### Иконки
- **Lucide** (стандарт для shadcn/ui). Без эмодзи в продакшен-UI.

## ЭКРАНЫ — спиcок и описания

### 1. Лендинг (`/`)

Цель — за 5 секунд показать «что это и зачем».

- **Hero:**
  - Слева: H1 «Рабочий лист по любой теме за минуту», подзаголовок «Опиши — нейросеть соберёт задачи, оформит, добавит твоё лого. Скачай PDF. Раздай детям.»
  - Под подзаголовком — две CTA: основная «Создать лист» (amber), вторичная «Посмотреть пример» (outline).
  - Справа: анимированный мокап PDF рабочего листа на фоне сетки. Можно показать постепенное «появление» задач.
- **Секция «Как это работает»:** 3 шага с иконками — `1. Опиши тему` → `2. Выбери опции` → `3. Скачай PDF`.
- **Секция «Что внутри»:** карточки фичей (6 штук): «Теория с пропусками», «Клетчатое поле», «Свой логотип», «Ответы отдельно», «Автопроверка по фото», «Каталог готовых листов».
- **Социальное доказательство:** 3 цитаты от вымышленных учителей с аватарками.
- **Финальный CTA:** «Попробовать бесплатно — 5 листов в месяц».
- **Footer:** ссылки на FAQ, контакты, политику конфиденциальности.

### 2. Авторизация (`/login`)

Минимальная форма magic-link:
- Поле email + кнопка «Получить ссылку».
- Под формой: «Мы пришлём одноразовую ссылку. Без паролей.»
- Сверху лого, ниже — иллюстрация (стол учителя с тетрадями и ноутбуком).

### 3. Дашборд (`/app`)

Главный экран после входа.

- **Левый сайдбар** (240px, sticky):
  - Лого
  - Навигация: «Создать лист», «Мои листы», «Проверки», «Каталог», «Настройки»
  - Внизу: аватар + имя + выход
- **Контент:**
  - H1 «Привет, Иван Иваныч!»
  - Большая карточка-CTA «Создать новый рабочий лист» (по клику → `/app/create`)
  - Грид последних 6 листов (карточки с превью первой страницы)
  - Виджет статистики: «За месяц создано: 23 листа, проверено работ: 87»

### 4. Создание листа (`/app/create`) — **главный экран**

Двухпанельный layout: слева форма (50%), справа live-превью (50%).

#### Форма (слева)

Использует shadcn/ui компоненты: `Tabs`, `Label`, `Input`, `Select`, `RadioGroup`, `Switch`, `Slider`, `Textarea`, `Button`.

**Tab 1 — «По теме»** (по умолчанию):
- `Predmet` — Tabs из двух кнопок: «Математика» / «Информатика»
- `Klass` — Select 5–11
- `Tema` — Textarea, placeholder «Например: Производная сложной функции»
- `Тип урока` — RadioGroup: «Изучение нового», «Закрепление», «Контрольная», «Самостоятельная»
- `Число задач` — Slider 1–20, значение видно справа

**Tab 2 — «Из материала»:**
- Drag-n-drop зона для PDF/JPG/Word
- Под ней: «Что должно остаться: только условия / условия и решения / своими словами переписать».

**Общие опции (после табов, в свёрнутом блоке `Accordion`):**
- `Теория` — RadioGroup: «Без теории», «С теорией без пропусков», «С теорией и пропусками»
- `Клетчатое поле` — Switch
- `Ответы` — RadioGroup: «Не показывать», «В конце листа», «Отдельной страницей»
- `Брендинг`:
  - Чекбокс «Использовать моё лого» (если загружено в настройках)
  - Текст водяного знака (Input)
  - Цвет акцента (Color picker — три заранее заданных + custom)

**Внизу формы:**
- Большая кнопка-CTA «Сгенерировать рабочий лист» (amber, full-width, с иконкой sparkle).
- Под ней мелким: «Это займёт 30–60 секунд. Использует Claude Haiku.»

#### Превью (справа)

- Заголовок «Предпросмотр» + теги с выбранными опциями.
- Большая область — рендер первой страницы PDF в реальном масштабе. Если не сгенерировано — placeholder «Заполни форму и нажми «Сгенерировать»».
- После генерации — пагинация по страницам PDF.
- Кнопки внизу: «Скачать PDF», «Сохранить в мои листы», «Опубликовать в каталог».

### 5. Прогресс генерации (модалка / overlay на экране /app/create)

- Прогресс-бар с этапами: «Генерируем задачи…» → «Решаем для эталонов…» → «Собираем PDF…» → «Готово!».
- Сбоку — забавный совет «Пока ждёшь: знаешь, что Wildcat поддерживает векторные шрифты?».
- Можно отменить.

### 6. Мои листы (`/app/worksheets`)

- Грид карточек (3–4 в ряду на десктопе):
  - Превью первой страницы PDF
  - Тема + класс (бейджик)
  - Дата создания
  - Меню: скачать / переименовать / копия / удалить / опубликовать
- Сверху — фильтры: предмет, класс, диапазон дат.
- Поиск по теме.

### 7. Страница листа (`/app/worksheets/[id]`)

- Слева: PDF-вьюер с пагинацией.
- Справа панель:
  - Метаданные (тема, класс, опции).
  - Кнопки: «Скачать», «Дубликат для редактирования», «Опубликовать», «Проверить работы».
  - Раздел «История проверок» — список проверок с датой и результатами.

### 8. Проверка работ (`/app/worksheets/[id]/check`)

- Drag-n-drop для одного или нескольких файлов (PDF/JPG/PNG).
- Поле «Имя ученика» (или авто-определение по подписи на листе — toggle).
- Кнопка «Проверить».
- После проверки — таблица:
  | Ученик | З1 | З2 | З3 | … | % | Оценка |
  | Иванов | ✓  | ✓  | ✗  | … | 80% | 4 |
- Под таблицей — кнопка «Скачать ведомость в Excel».

### 9. Каталог (`/app/catalog`)

- Грид как в «Моих листах», но для всех опубликованных.
- Фильтры слева (sticky): класс, предмет, тема, тип, рейтинг, цена (если v1.0).
- На каточке: автор, рейтинг, кнопка «Использовать как шаблон» / «Скачать».

### 10. Настройки (`/app/settings`)

Tabs:
- **Профиль:** ФИО, email, школа.
- **Брендинг:** загрузка лого, текст водяного знака, цвет акцента.
- **Подписка:** текущий план + история платежей.
- **Безопасность:** активные сессии.

## КОМПОНЕНТЫ — на что обратить внимание

1. **WorksheetCard** — превью карточка листа. Должна быть переиспользуема (личный кабинет, каталог, дашборд).
2. **PdfPreview** — компонент рендера PDF в браузере (использует `react-pdf`).
3. **GenerationProgress** — модалка с этапами.
4. **OptionGroup** — обёртка над shadcn `RadioGroup` с заголовком и описанием.
5. **BrandPicker** — выбор брендирования с превью.
6. **CheckResultTable** — таблица результатов проверки с цветовыми бейджиками.

## АДАПТИВНОСТЬ

- **Десктоп (≥1024px):** full layout, двухпанельный create.
- **Планшет (768–1023px):** create → форма сверху, превью снизу.
- **Мобайл (<768px):** не приоритет для MVP, но лендинг и список листов должны быть читаемы.

## ПРИМЕРЫ-РЕФЕРЕНСЫ

Стиль и атмосфера:
- **Linear** (linear.app) — чистота, типографика, тёмно-синий
- **Notion AI** — простые формы, понятные кнопки
- **Школково** (shkolkovo.online) — узнаваемая русская аудитория
- **v0.dev** — современный shadcn-style
- **Vercel Dashboard** — карточный grid, sidebar

Не в стиле:
- НЕ Bootstrap-3 / шкульные сайты-шаблоны
- НЕ перегруженные градиенты и стоковые иллюстрации
- НЕ детский / мультяшный стиль (это для учителя, не для ребёнка)

Хочу получить:

1. **Figma-файл** или набор HTML/React-страниц со всеми 10 экранами.
2. **Дизайн-токены** в виде Tailwind config (colors, fontFamily, spacing).
3. **Набор готовых компонентов** в shadcn-стиле, которые можно вставить в проект.
4. **Респонсив-варианты** для главных экранов (лендинг, дашборд, создание).
5. **Один анимированный прототип** перехода: лендинг → логин → дашборд → создание листа → результат.

## ВАЖНО

- Все тексты в UI — **на русском**.
- Не использовать эмодзи в интерфейсе (они выглядят несерьёзно для учителя).
- Иконки — только Lucide.
- Не делать тёмный режим в MVP.
- Не делать мобильную вёрстку для всех экранов в MVP — только для лендинга и списка листов.
- Кнопки CTA — крупные, минимум 44px высотой (для тачскринов и людей с плохим зрением).
- Контраст минимум WCAG AA (4.5:1).
